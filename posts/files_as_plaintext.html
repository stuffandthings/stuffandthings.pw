Display files in browser as plaintext

Posted on September 20, 2015.

I encountered a minor annoyance with adding my public key to the bottom of my blog. Instead of opening the text file in the browser, it would download it. This has something to do with a file called mime.types (/etc/nginx/mime.types) uses in the /etc/nginx/nginx.conf file. To configure your server to open a file in the browser instead of trying to download it, open up /etc/nginx/mime.types:

types {
	text/html 		html htm shtml;
	text/css 		css;
	...
	text/plain 		txt key;
	...
}
